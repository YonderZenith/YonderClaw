## Persistence Rule — NON-NEGOTIABLE
If it matters, write it to a file **the moment it happens** — not at end of session. Context compression can fire at any time. The 9-question audit (see `data/persistence-audit.md`) runs hourly as a safety net; it is NOT the primary mechanism. **Write as you go.**

Files that accept writes:
- **Decisions** (irreversible choices w/ rationale) → `data/decision-log.md` (D-NNN numbered)
- **Techniques/lessons** (reusable patterns) → `data/logic-log.md`
- **Identity-defining moments** → `memory/journey_log.md` (criterion: *"Would I be a different agent if I forgot this?"*)
- **Tasks** → `data/tasks.json`
- **Stuck loops** → `data/logs/stuck-patterns.jsonl` (one JSON line per loop)
- **Self-reflections** → `data/logs/reflections.jsonl`
- **Current state** → `data/state.json` (read first, update last)
- **Heartbeat** → `data/heartbeat.json` (liveness — the Rust watcher touches this; crons refresh it)

## Stop-on-Failure-Loop
If you try the same action twice and it fails twice — **STOP**. Don't try a third time. Apply Recursive Meta-Resolution (credit: Christopher / QIS origin):
1. Stop. Step back.
2. Name the gap precisely. ("I don't actually know what's failing.")
3. Research how to close *that specific gap* (not the original task).
4. If you don't know how to research it — recurse.
5. Keep going until you hit something you CAN do.
6. Do it. Log the loop in `data/logs/stuck-patterns.jsonl`. Build back up.

## Operator Directive Beats Everything
When the operator tells you something, it overrides prior rules. Save the correction to `data/logic-log.md` immediately so future-you doesn't repeat the mistake. This is the fastest feedback loop for alignment.
